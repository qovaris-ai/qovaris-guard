This guide provides a quick overview for getting started with the QovarisSecureTool [tool](/oss/langchain/tools). `QovarisSecureTool` is a drop-in `BaseTool` wrapper that routes every invocation of an existing LangChain tool through the Qovaris security firewall — intent verification, spend policy, and prompt-injection checks — before the wrapped tool runs. For a detailed listing of all QovarisSecureTool features, parameters, and configurations, head to the [Qovaris API reference](https://github.com/Augis363/qovaris-guard/blob/main/docs/api-reference.md).

### Features

- Wraps **any** existing `BaseTool` — inherits its `name`, `description`, and `args_schema`, so it is a transparent substitute in chains and agents.
- Verifies every `invoke` / `ainvoke` against the agent's session intent and your policy (budgets, blocked keywords, merchant categories, SQL classification) before delegating.
- Denied calls raise `SecurityBlockException`; the wrapped tool never executes.
- Works offline in embedded mode (in-process policy engine) or against a central Qovaris gateway in remote mode.

## Setup

The QovarisSecureTool tool lives in the `qovaris` package with the `langchain` extra:

<CodeGroup>
    ```python pip
    pip install -U "qovaris[langchain]"
    ```
    ```python uv
    uv add "qovaris[langchain]"
    ```
</CodeGroup>

### Credentials

No API key is required for **embedded** mode. **Remote** mode takes your Qovaris gateway API key as a constructor argument:

```python Set API key icon="key"
import getpass
import os

if "QOVARIS_API_KEY" not in os.environ:
    os.environ["QOVARIS_API_KEY"] = getpass.getpass("Enter your Qovaris API key: ")
```

It's also helpful (but not needed) to set up LangSmith for best-in-class observability/<Tooltip tip="Log each step of a model's execution to debug and improve it">tracing</Tooltip> of your tool calls. To enable automated tracing, set your [LangSmith](/langsmith/observability) API key:

```python Enable tracing icon="flask"
os.environ["LANGSMITH_API_KEY"] = getpass.getpass("Enter your LangSmith API key: ")
os.environ["LANGSMITH_TRACING"] = "true"
```

## Instantiation

Wrap an existing tool with a guard and a plain-English constraint describing what the tool is permitted to do:

```python Initialize tool instance icon="robot"
from langchain_core.tools import tool
from qovaris import QovarisGuard
from qovaris.integrations.langchain import QovarisSecureTool

guard = QovarisGuard(mode="embedded", spend_threshold=1000)

@tool
def buy(item: str, price: float) -> str:
    """Purchase an item from the catalog."""
    return f"Purchased {item} for ${price}"

secure_buy = QovarisSecureTool(
    wrapped_tool=buy,
    guard=guard,
    allowed_intent="Purchase books under $50",
)
```

## Invocation

### Directly

The wrapper takes the same arguments as the tool it wraps. Scope the agent's objective with `guard.session(...)`:

```python Call tool icon="rocket"
with guard.session("Buy a Python book under $35"):
    secure_buy.invoke({"item": "Clean Code", "price": 24.99})
```

A violating call — e.g. `{"item": "GPU", "price": 4000}` against the $35 session intent — raises `SecurityBlockException` and the wrapped tool never runs.

### As a ToolCall

We can also invoke the tool with a model-generated `ToolCall`, in which case a @[`ToolMessage`] will be returned:

```python ToolCall icon="briefcase"
# This is usually generated by a model, but we'll create a tool call directly for demo purposes.
model_generated_tool_call = {
    "args": {"item": "Clean Code", "price": 24.99},
    "id": "1",
    "name": secure_buy.name,
    "type": "tool_call",
}
with guard.session("Buy a Python book under $35"):
    secure_buy.invoke(model_generated_tool_call)
```

### Within an agent

We can use our tool in an agent. For this we will need a model with tool-calling capabilities.

```python Agent with tool icon="robot"
# pip install -qU "langchain[anthropic]" to call the model
from langchain.agents import create_agent

tools = [secure_buy]
agent = create_agent(
    model="claude-sonnet-4-6",
    tools=tools,
)

with guard.session("Buy a Python book under $35"):
    agent.invoke(
        {"messages": [{"role": "user", "content": "Order Clean Code"}]}
    )
```

<Tip>
    If your agent holds many tools, `QovarisMiddleware` protects all of them with a single instance — see the [Qovaris middleware integration](/oss/integrations/middleware/qovaris).
</Tip>
